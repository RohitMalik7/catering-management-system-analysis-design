# Catering Management System – Systems Analysis and Design

> **Author:** Rohit Malik | **Domain:** Systems Analysis | **Type:** Academic Project | **Purpose:** Portfolio & Educational

---

## Overview

This project presents a full systems analysis and design for a catering services platform called **OnceForAll**, covering end-to-end system modelling including requirements engineering, UML diagrams, use case design, and user acceptance testing.

The system connects customers with caterers, enabling job requests, quote management, payment processing, feedback collection, and business analytics.

---

## Project Objective

The objective of this project is to design a scalable system that manages all interactions between customers and caterers, including job requests, quote handling, payments, and feedback mechanisms -from initial requirements through to validated system models.

---

## Key Components

### 1. Stakeholder Analysis

Identified and classified key stakeholders across the system:

| Stakeholder | Role |
|---|---|
| Customers | Submit job requests and provide feedback |
| Caterers | Receive requests, submit quotes, and deliver services |
| System Administrators | Manage platform operations and user access |
| Business Owner | Oversees platform performance and business rules |
| Payment Processors | Handle financial transactions between parties |
| Investors and Regulators | Monitor compliance and business performance |

### 2. Requirements Engineering

- Functional requirements covering job requests, caterer matching, quote management, and notifications
- Non-functional requirements defined using the FURPS+ framework (performance, security, usability, scalability)

### 3. Use Case Modelling

- User Goal Technique applied to identify core use cases
- Event Decomposition used to capture system-triggered and time-based events
- Detailed use case descriptions developed, including the "Find Caterer" use case

### 4. UML Modelling

| Diagram Type | Purpose |
|---|---|
| Domain Model Class Diagram | Models entities, attributes, and relationships |
| System Sequence Diagram | Captures interactions between actors and the system |
| Activity Diagram | Represents workflow and decision logic |
| State Machine Diagram | Tracks object states and transitions across the system |

### 5. Data and Process Modelling

- CRUD Matrix developed to map system operations against entities
- Entity interaction mapping to track data flow across system components

### 6. Testing and Validation

- User Acceptance Test Plan developed with defined test conditions
- Expected outcomes documented for core system scenarios
